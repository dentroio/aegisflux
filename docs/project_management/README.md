# AegisFlux Project Management

This directory is the active project-management home for AegisFlux planning and executable work orders.

## Current Direction

The active product loop is:

**Discover -> Explain -> Design -> Simulate -> Govern -> Adapt**

The current growth wave is tracked in the Product Platform work orders:

- `WO-GROWTH-001`: ABOM fleet insights and change detection.
- `WO-GROWTH-002`: Evidence graph UX and explainability.
- `WO-GROWTH-003`: Control simulation depth and decision history.
- `WO-GROWTH-004`: Agent health and readiness scoring.
- `WO-GROWTH-005`: Adaptive detection workflow maturity.
- `WO-GROWTH-006`: First-value demo polish and sample scenarios.
- `WO-GROWTH-007`: Audit-mode enforcement adapter foundation.

The remaining work is now tracked as an operational-readiness wave:

- `WO-OPS-001`: End-to-end pipeline validation.
- `WO-OPS-002`: Lab agent connectivity and heartbeat reliability.
- `WO-OPS-003`: Service health, discovery, and resilience.
- `WO-OPS-004`: Ingest and ETL hardening.
- `WO-OPS-005`: Observability, metrics, and logging baseline.
- `WO-OPS-006`: Security and service authentication baseline.
- `WO-OPS-007`: Performance, load, and resource baseline.
- `WO-OPS-008`: Deployment, release, and operator readiness.

After operational readiness, the AI-native leap is tracked as a next capability wave:

- `WO-AGENTS-001`: AI agent harness and tool runtime.
- `WO-AGENTS-002`: Evidence-bound reasoning contract.
- `WO-GOV-001`: Agent governance, memory, and decision ledger.
- `WO-AGENTS-003`: Endpoint Analyst deep agent.
- `WO-ADAPT-001`: Detection opportunity research agents.
- `WO-ADAPT-002`: Detection candidate simulation harness.
- `WO-CONTROL-002`: Control Design Copilot.
- `WO-FLEET-001`: AI capability drift radar.
- `WO-ENFORCE-001`: Enforcement readiness scorecard.
- `WO-DEMO-002`: Autonomous demo scenario generator.
